# 给联网检索 AI 的填写说明（可整份复制）

把**本文件全文**发给能联网搜索的 AI，并在末尾补上你的主题。对方应只输出 **一个** Markdown 文件正文（不要解释过程，不要输出多个文件）。

---

## 你的任务

根据用户给出的主题（小说 / 电视剧 / 历史人物 / 历史时期），联网核实后，生成一份**既定事实素材笔记**，供写作 Agent 的资料库检索使用。

## 硬性要求

1. **只产出 1 个 `.md` 正文**（单文件）。除非用户明确要求拆成多文件，否则禁止拆成 overview/characters 多个文档。
2. 使用下方「输出模板」结构；可增删 `##` 小节，但必须保留元数据头与 `## 概要`。
3. 标题只用 `#` / `##` / `###`（不要用四级标题当主结构）。`##`/`###` 标题里尽量带**专名**，便于检索。
4. 写可核对的事实、设定、人物关系、时间线要点；允许较长（数千至上万字均可）。
5. **禁止**大段粘贴小说/剧本受版权保护的原文；用复述与归纳。
6. **禁止**输出任何脚注/引用记号，包括但不限于：`[reference:0]`、`[1]`、`[^1]`、`（见某某）` 式联网脚注。正文只要干净事实叙述；来源核对自行完成，不要写进文件。
7. 不确定的内容标注「通行说法」或省略；不要编造具体引文页码装权威。
8. 文首元数据按主题填写：
   - 小说 → `类型: novel`，路径暗示 `writing/novels/`
   - 电视剧 → `类型: drama`
   - 人物 → `类型: person`
   - 时期 → `类型: period`
9. `主题slug`：小写英文/拼音 + 连字符（如 `liang-jian`、`hong-lou-meng`）。在回复最开头单独一行写：  
   `建议文件名: <slug>.md`  
   然后再输出 Markdown 正文。人类保存时**删掉这一行**，只保留从 `#` 开始的正文；文件请按建议文件名保存，不要用 `drama1.md` 这种占位名。

## 输出模板（按此填写）

```markdown
# <正式名>

> 类型: novel
> 体裁: volume
> 别名: <异名，逗号分隔>
> 时期: <时代或虚构背景>
> 主题slug: <slug>
> 分册: 单篇
> 来源说明: 演示用既定事实资料，非学术论文；请勿粘贴受版权保护的大段原文

## 概要

（一段完整介绍：作品定位、时代、核心冲突、为何重要。专名写全称。）

## 世界观与背景

（可长。制度、地理、势力、科技/武力规则等。）

## 人物

### <人物正式名>

（身份、性格要点、与主线的关系。专名反复写出，少用「他/其」。）

### <另一人物>

## 主线剧情

### <卷/部/阶段名或前期>

（复述情节要点，可详，勿原文。）

### <中期 / 后期>

## 时间线要点

- （可选列表：关键节点）

## 可引用细节

- （称谓、物件、名场面要点、易写错的设定）
- …

## 勿混淆

- （与续作、同名作品、衍生、常见误传的区别）
```

人物主题把「世界观 / 主线剧情」换成生平阶段即可；时期主题用政治/军事/社会等 `##`；电视剧同小说模板，剧情可按季/集阶段划分。
