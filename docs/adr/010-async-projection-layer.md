# ADR-010: 异步任务与投影层不阻塞主执行路径

## 状态

已接受（2025-06-30）

## 背景

`agent-langraph` 在单进程 lifespan 中初始化调度器、记忆写回、评估采样等，并与 Turn 主链路共享故障域。若新项目将类似能力塞回 `runtime` 主 loop 或同步 API，将重复「启动慢、错误边界大、主路径延迟高」的问题。

需要为**非用户可感知闭环**的副作用指定正式归属。

## 决策

1. 定义逻辑层 **异步任务与投影层**（物理上 Phase 0 可内聚于 `api` 模块，Phase 2 起可拆 worker + redis）。
2. **必须移出 Turn 主路径**的能力包括但不限于：
   - 记忆写回
   - 检索索引更新与 embedding 构建
   - transcript 压缩归档
   - 评估采样
   - usage 统计聚合
   - 通知推送
   - projection 补算与全量重建
   - 审计汇总
3. **主路径保留**：输入编译、上下文组装、模型调用、工具执行、终止判断、事件追加、checkpoint、最小 domain 写入。
4. projection 刷新失败 **不得** 阻断 Turn 执行闭环或丢失 turn 主状态。
5. 异步触发优先通过 **outbox**（`outbox_jobs` 表）或已持久化事件，保证可重试与可观测。
6. `runtime` **禁止**内嵌长期后台线程完成上述工作；`api` **禁止**同步执行长时间 CPU / 模型 / 全库扫描任务。

### Phase 演进

| 阶段 | 承载方式 |
|------|----------|
| Phase 0 | `api` 内投影刷新模块；事件已落库 |
| Phase 1 | outbox 表 + 重试语义 |
| Phase 2 | `redis` profile + worker 容器 |

## 理由

- 缩短用户可感知延迟，与 ADR-005 最短主路径原则一致
- 降低故障耦合：索引重建失败不应导致对话不可用
- outbox 使副作用可审计、可重放，优于 fire-and-forget 线程
- 为 Phase 4 运营面（配额、灰度）预留标准扩展点

## 后果

### 正面

- runtime 专注执行强度，api 保持相对轻量
- 日志可区分主链路与异步链路，排障更清晰
- worker 可独立扩缩，不拖累 Turn 延迟

### 负面

- 投影相对事件可能短暂滞后，前端需容忍最终一致
- outbox 与 worker 引入额外运维组件（Phase 2+）
- 需明确每个异步任务的失败归宿与告警策略

## 备选方案

| 方案 | 否决原因 |
|------|----------|
| 全部同步在 Turn 结束时做完 | 尾延迟高，工具化后更明显 |
| runtime 内 BackgroundTasks 无 outbox | 进程崩溃丢任务，不可重放 |
| 独立 10+ 微服务处理副作用 | Phase 0–2 过度设计 |
| 不做 projection，仅 SSE | 重连与列表页体验差 |
