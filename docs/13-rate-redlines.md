## R1–R5（所有改进的前置约束）

| # | 原则 | 验收含义 |
|---|------|----------|
| **R1** | 不挡 `turn.accepted` | 新逻辑不得推迟 TTFB |
| **R2** | 首 token 前不加模型调用 | 不为「正确」再问一轮同步 LLM |
| **R3** | 热路径同步 CPU ≤ 毫秒级 | schema/正则 OK；重 tokenizer / CE **默认不进**热路径 |
| **R4** | 重活异步 / 离线 / 用户触发 | 索引、评分、核查不上用户等待 |
| **R5** | 可测才合并 | 单测或 golden；延迟相关须有断言或 SLO 对照 |

**默认合并：** 先全绿；带超时/降级的项须可观测；不引入未改写的阻塞型方案。
